# User Manual

## Guest

1. Open the home page.
2. Enter part of a temple name and/or location and select **Search**.
3. Select **Book Darshan** on a temple card.
4. Review temple hours, description, amenities, events, ratings, and available dates/slots.
5. Use the floating audio control to mute/unmute the temple track.

Guests can browse but must sign in to book, donate, or review.

## Create an account and sign in

1. Select **Register**.
2. Enter name, email, password (the UI requires at least six characters), optional phone/address, and solve the math prompt.
3. Select **Register**. A successful response signs you in automatically.
4. Returning users use **Login** with email/password.

Security note: the current form displays role choices, including administrator. That is an implementation flaw, not a recommended user workflow. Production registration must always create a normal devotee account; privileged accounts must be provisioned by an authorized administrator.

## Devotee booking

1. Sign in with a `USER` account.
2. Open a temple and choose a future slot with enough seats.
3. Select **Book Now** and choose the number of devotees.
4. Enter each devotee’s name, age (UI accepts 1–120), and gender.
5. Submit to create a five-minute seat hold.
6. On the simulated checkout screen, enter a 16-digit card number, a non-expired month/year, and a three-digit CVV.
7. Confirm before the timer expires.
8. Print the generated ticket. The ticket includes temple, slot, devotees, amount, booking status, and a `DE-` ticket code.

No real payment is charged. Card data is validated in the browser and is not included in the backend confirmation request.

## User dashboard

Open **Dashboard** from the navigation menu.

- **My Bookings:** view booking status and ticket details.
- **My Contributions:** view recorded donation receipts.
- **Profile Settings:** update name, email, phone, address, or password.

Although a cancellation handler exists in the source, the current booking-list UI does not expose a cancellation button. Cancellation is available through the API and has a known seat-restoration defect for pending holds.

## Donations

1. Open **Donations**.
2. Sign in if prompted.
3. Select a temple, preset/custom INR amount, purpose, and donor name or anonymous option.
4. Complete the simulated card-format fields.
5. Submit and print the donation receipt.

The generated transaction record is a demo record, not proof of bank settlement or a tax receipt.

## Reviews

On a temple detail page, an authenticated user can choose a rating, enter a comment, and submit. Submitting again updates that user’s existing review for the same temple. Seeded anonymous-style ratings without a user reference remain separate.

## Organizer dashboard

- **Slot Scheduler:** select a temple and enter date, time range, seats, and price; create or delete slots.
- **Bookings Manager:** view booking records.
- **Notifications:** view demo-only in-memory notifications; they are not generated by the backend.
- **Profile Settings:** update organizer profile.

Current organizers are not assigned to specific temples and can access global organizer data. Do not use this authorization model in production.

## Admin dashboard

- **Analytics:** user/organizer/temple totals, booked counts, simulated revenue, temple popularity, and age demographics.
- **Users:** filter by role and delete accounts.
- **Temples:** create and delete temples.
- **Events:** schedule events.
- **Facility:** log maintenance requests and update their statuses.

The API also supports user updates, temple updates, and event deletion, but the current dashboard does not provide all of those controls.

## Troubleshooting

| Symptom | Likely cause | Action |
|---|---|---|
| “Database is offline” / 503 | MongoDB not connected | Start MongoDB or correct `MONGODB_URI`; restart server if needed |
| Client cannot reach API | Backend stopped or fixed localhost URL unsuitable | Start port 5000 server; configure environment-based API URL |
| 401 | Token missing, invalid, or expired | Log out and sign in again |
| 403 | Account role lacks permission | Use the correct authorized account; do not bypass the API |
| No future slots | Seed slots passed or organizers have not added slots | Create new future slots |
| Hold expired | Five-minute window elapsed | Close and start a new booking |
| Dashboard appears but data fails | Frontend routes lack role guards | Use the dashboard matching the authenticated role |
